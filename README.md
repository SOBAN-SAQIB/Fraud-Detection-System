# Real-Time Financial Fraud Detection

A streaming machine-learning pipeline that detects fraudulent financial
transactions in real time. A Gradient-Boosted Trees model is trained on the
**PaySim** synthetic transaction dataset with **PySpark**, then served over a
**Kafka → Spark Structured Streaming** pipeline whose predictions feed a live
web dashboard.

Built for the Big Data Analytics course (**CSDS4473**).

---

## Architecture

```
                    ┌──────────────────┐
  paysim_data.csv → │  train_pipeline  │ → fraud_detection_model/  (saved GBT pipeline)
                    │      .py         │ → test_stream_source/     (unlabeled test rows)
                    └──────────────────┘

  test_stream_source/  ──►  producer.py  ──►  Kafka topic: "transactions"
                                                      │
                                                      ▼
                                              ┌──────────────────┐
                                              │   processor.py   │  Spark Structured Streaming
                                              │  (loads model,   │  reads Kafka, scores each txn
                                              │   scores stream) │
                                              └──────────────────┘
                                                      │
                                                      ▼
                                       data/shared/live_logs.json (latest 40 predictions)
                                                      │
                                                      ▼
                                       ui/index.html  →  http://localhost:8080
                                       (polls the JSON every 500 ms, Chart.js dashboard)
```

## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) and Docker Compose
- The PaySim dataset (download below)

## Getting the dataset

The dataset is too large to ship in the repository. Download the **PaySim**
synthetic financial dataset from Kaggle:

- https://www.kaggle.com/datasets/ealaxi/paysim1

Then place it at the path below, renamed to `paysim_data.csv`:

```
datasets/paysim_data.csv
```

The training script expects these columns:
`step, type, amount, nameOrig, oldbalanceOrg, newbalanceOrig, nameDest,
oldbalanceDest, newbalanceDest, isFraud, isFlaggedFraud`.

## Running the project

### 1. Start the stack

```bash
docker compose up -d --build
```

This launches Zookeeper, Kafka (`localhost:9092`), the nginx UI server
(`localhost:8080`), and the PySpark app container.

### 2. Train the model

```bash
docker compose exec app-pipeline spark-submit train_pipeline.py
```

This produces `fraud_detection_model/` (the saved pipeline) and
`test_stream_source/` (clean, unlabeled rows used to simulate the live feed).

### 3. Start the streaming processor

In one terminal:

```bash
docker compose exec app-pipeline spark-submit processor.py
```

It loads the model and waits for transactions on the Kafka `transactions` topic.

### 4. Start the producer

In a second terminal:

```bash
docker compose exec app-pipeline python producer.py
```

It reads rows from `test_stream_source/` and publishes them to Kafka, driving
the live stream.

### 5. Open the dashboard

Visit **http://localhost:8080** to watch transactions being scored in real time.

## How it works

### Feature engineering

Both training and streaming derive the same features so the model sees a
consistent input shape:

- `hour_of_day` = `step % 24`
- `day` = `step / 24`
- `is_transfer_cashout` = 1 for `TRANSFER` / `CASH_OUT`, else 0
- `balance_change` = `|oldbalanceOrg - newbalanceOrig|`
- `amount_ratio` = `amount / (oldbalanceOrg + 1)`
- `typeIndex` = `StringIndexer` encoding of the transaction `type`

### Model

- **GBTClassifier** — `maxIter=20`, `maxDepth=5`, `stepSize=0.1`, `seed=42`
- Pipeline: `StringIndexer -> VectorAssembler -> GBTClassifier`
- 70 / 15 / 15 train / validation / test split
- Severe class imbalance is handled by **upsampling the fraud (minority) class**
  with replacement before training
- Evaluation: ROC-AUC, weighted precision, weighted recall, and F1 are printed
  on the validation set, plus ROC-AUC on the held-out test set

## Troubleshooting

- **`producer.py` must be implemented.** It needs to read rows from
  `test_stream_source/` and publish them to the `transactions` topic on
  `kafka:29092`, using the schema in `processor.py`
  (`transaction_id, step, type, amount, oldbalanceOrg, newbalanceOrig,
  oldbalanceDest, newbalanceDest`). If it's still a stub, the dashboard will
  stay empty because nothing reaches Kafka.
- **Stream won't restart after a code change.** Spark resumes from
  `spark_checkpoints/`. If you change the schema or feature logic and hit a
  restart error, delete the `spark_checkpoints/` folder to start fresh.
